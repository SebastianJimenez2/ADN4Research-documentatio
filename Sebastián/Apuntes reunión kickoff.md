## Stakeholder 2 #PAMELA
### Herramientas que ha usado:
- Atlas.ti
- Rayyan
### Proceso que realiza:
Guía de Barbara Kitchenham
Tres fases:
- Fase de búsqueda: cadenas, fuentes de información, alcance, preguntas.
- Fase de ejecución: ejecución de cadenas de búsqueda, refinamiento de las cadenas, proceso iterativo, dependiendo de la búsqueda se mejora cadena y pregunta de investigación.
- Fase de extracción: numero de artículos en donde se extrae la información.
Problemas:
- Elaboración de cosas manuales.
- Búsqueda oportunista => ppig, ibm
- Referencias, snowballing
>[!Note]
>Hizo una investigación sola, entonces fue importante tener un par, al menos debe hacer dos para tener una forma de contrastar.

# ¿En que fases del proceso RSL encuentra problemas?
## Stakeholder 1 
- Poder ordenar la información encontrada.
- Se olvidan en dónde estaba la información.
- Existen fuentes que no se pueden descargar.
- Trasladar las referencias hacia un archivo latex, si no hay una  buena consolidación de referencia es posible que se pierdan, o es complejo poder alcanzar.
- No se puede hacer relación entre las bibliografías encontradas por referencia de los paper, mapas de conexión, a veces es mejor que las cadenas de búsqueda.
## Stakeholder 2
- Documentar todos los pasos previos hasta llegar a la cadena final, trazabilidad.
- Cada motor de búsqueda mantiene un estándar distinto.
- Si existe actualización de criterio no hay trazabilidad en la fase de ejecución.
- LA TRAZABILIDAD ES IMPORTANTE PORQUE DA RIGOROSIDAD AL PROCESO. Es hacer un proceso reproducible.
- Un trabajo debe cumplir con cuatro aspectos
	- Cualitativo: 1. Confiabilidad, lo que sacas como resultado viene de algún lado, demostrando en proceso de transición entre el inicio y el resultado.
## Stakeholder 3
- Proceso manual tedioso, se ven forzado a hacerlo
- Cadena de búsqueda tiene que probar en diferentes fuentes y no es compatible y tiene que escribir para cada motor de diferente forma, si hay un cambio es tedioso. Las cadenas cambian un montón de un buscador a otro.
- Herramienta que verifique si la cadena es muy dispersa
- La descarga de la información es complicada, se debe buscar el paper cuando no se puede descargar
- Al repartir las cosas se encuentra problemas, la división del estudio
- Atlas no permite visualizar las notas entre investigadores
- Proceso debe poder estar empaquetado, debe dar un resumen de todo el proceso, cada paso o cambio realizado, para que sea reproducible y transparente.
